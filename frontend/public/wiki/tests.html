4. Prochaine étape — Tests automatisés (Jest + Supertest)

📦 Installation (à faire une fois) :
Terminal
npm install --save-dev jest supertest

# Puis ajouter dans package.json :
# "test": "jest",
# "test:watch": "jest --watch",
# "test:coverage": "jest --coverage"

Les fichiers de test iront dans tests/api.test.js, tests/validation.test.js, etc.

🎯 Suite de Tests Interactive

Interface web complète pour tester automatiquement tous les aspects du site.

🧪 Ouvrir la Suite de Tests

Nouvelle fenêtre 17 tests

🌐 URL directe 
http://localhost:3000/scripts/maintenance/test-suite.html
✨ 17 tests automatisés

Organisés en 3 priorités : Critique (P1), Important (P2), Recommandé (P3)

🔴 Tests Priorité Critique (P1)

  • Images & Assets - Vérifie que toutes les images produits existent
  • Fiches HTML - Vérifie l'accessibilité des fiches produits
  • Base de données - Contrôle l'intégrité des données
  • API Produits - Teste tous les endpoints API
  • Catégories - Vérifie que chaque catégorie a des produits

🟡 Tests Priorité Importante (P2)

  • Liens internes - Vérifie tous les liens du site
  • Performance - Mesure les temps de chargement
  • Responsive Design - Test mobile/tablette/desktop
  • Système de cache - Vérifie localStorage
  • Recherche - Teste la fonctionnalité de recherche
  • Filtres - Vérifie les filtres par catégorie

🔵 Tests Priorité Recommandée (P3)

  • SEO - Balises meta, titres, descriptions
  • Accessibilité - Navigation clavier, contrastes
  • Sécurité - Tests injection SQL/XSS basiques
  • Validation HTML - Conformité W3C
  • Erreurs console - Détection JavaScript
  • CORS - Configuration cross-origin
  • Service Worker - Fonctionnement SW
  • Formulaires - Validation des champs
  • Format images - Taille et compression
  • JSON API - Validité des réponses
  • Top du mois - Filtrage des tops
  • Pagination - Si implémentée

🖥️ Tests via Scripts Node.js

Vérification des Assets

Terminal 
node scripts/maintenance/verify-assets.js

Vérifie :

  • ✅ Toutes les images référencées existent physiquement
  • ⚠️ Incohérences de noms (ex: "apple-" manquant)
  • 🗑️ Images orphelines (fichiers non utilisés)

Vérification de la Base de Données

Terminal 
node scripts/maintenance/verify-database-state.js

Vérifie :

  • Intégrité des données produits
  • Champs manquants ou vides
  • Formats incorrects (prix, liens, etc.)
  • Cohérence des catégories

Tests Unitaires Jest

Terminal 
# Tous les tests
npm test

# Mode watch (relance auto)
npm run test:watch

# Avec couverture de code
npm run test:coverage

Fichiers testés :

  • tests/api.test.js - Endpoints API
  • tests/database.test.js - Connexions DB
  • tests/generation.test.js - Génération de fiches

⚡ Tests de Performance

Google Lighthouse

Dans Chrome DevTools (F12) :
  1. Onglet Lighthouse
  2. Sélectionne les catégories : Performance, Accessibilité, SEO, Best Practices
  3. Mode : Desktop ou Mobile
  4. Clique sur "Analyze page load"

Scores recommandés :

  • 🟢 Performance : >90
  • 🟢 Accessibilité : >90
  • 🟢 SEO : >90
  • 🟢 Best Practices : >90

Network Throttling

Simule une connexion lente :

  1. DevTools → Network
  2. Dropdown : "No throttling" → Fast 3G / Slow 4G
  3. Recharge la page

❓ FAQ Tests

Combien de temps prend la suite complète ?

Environ 2-3 minutes pour les 17 tests automatisés.

Puis-je arrêter les tests en cours ?

Oui, bouton "⏸️ Arrêter" dans l'interface test-suite.

Les tests modifient-ils les données ?

Non, tous les tests sont en lecture seule. Aucune modification de la base de données.

Quels tests lancer quotidiennement ?

Tests Priorité Critique (P1) - 5 tests en 30 secondes.

Comment corriger les erreurs détectées ?

Chaque test affiche un message détaillé. Utilise les scripts de maintenance pour corriger automatiquement certains problèmes.
